## Key Optimizations

### 1. Schema Validator Cache (16.8x speedup)

**Before**: 每次 `validate_input/validate_output` 调用都重新解析 JSON Schema 字符串 + 编译 jsonschema Validator。
**After**: 全局 `VALIDATOR_CACHE`（`RwLock<HashMap<u64, Arc<Validator>>>`），以 schema hash 为 key 缓存编译后的 validator。首次编译 ~6µs，后续命中 ~377ns。

**Impact**: 生产环境中 schema 极少变更，绝大多数调用走缓存。热路径延迟降低 94%。

### 2. Store Stats SQL N+1 Elimination (1.55x speedup)

**Before**: 每个 rune 执行 4 条独立 SQL + 1 次全表 latency 读取，N 个 rune = 4N+1 条查询。
**After**: 2 条 SQL：1 条 GROUP BY 聚合 + 1 次有序扫描（流式分组计算 P95），应用层合并。

**Impact**: stats_enhanced 从 115µs 降到 74µs。10 个 rune 时差距更大（原方案 41 条 SQL vs 2 条）。

### 3. Regression Tests Added

- 3 个 schema cache 正确性测试（缓存命中一致性、多 schema 独立缓存、input/output 共享）
- 3 个 store stats 正确性测试（精确数值验证、空表边界、大规模验证）
