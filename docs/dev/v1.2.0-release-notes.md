# Rune v1.2.0 Release Notes

> 主题：生产加固 + 可观测性 + 扩缩容

---

## 新增功能

### P0 — 生产必需

- **模块 A：重试与断路器** — 远程 Caster 调用自动重试（指数退避，可配置 `max_retries`/`base_delay_ms`/`max_delay_ms`），per-Caster 断路器（Closed/Open/Half-Open 三态，可配置 `failure_threshold`/`success_threshold`/`reset_timeout_ms`）
- **模块 B：请求追踪** — 统一 `request_id` 贯穿 HTTP→Relay→Caster 全链路，W3C Trace Context（`traceparent`/`tracestate`）提取与透传，Flow `parent_request_id` 关联，错误响应携带 `request_id`
- **模块 C：Store 并发优化** — SQLite 读写分离连接池（`reader_pool_size` 默认 4），API Key 三阶段驱逐缓存（正/负查 TTL 可配置），性能回归验证通过
- **模块 D：Flow 持久化** — `flows` 表 Store CRUD，Gate write-through 层，启动时自动加载已保存 Flow

### P1 — 功能增强

- **模块 E：Admin Key Bootstrap** — `rune key bootstrap` CLI 命令 + `RUNE_AUTH__INITIAL_ADMIN_KEY` 环境变量，首次启动导入管理员 Key，幂等安全
- **模块 F：细粒度 Rate Limiting** — SHA-256 rate key 隔离，per-Rune limiter（`rate_limit.per_rune` 配置），Caster 并发上限（`default_caster_max_concurrent` 默认 1024）
- **模块 G：Caster 健康检查** — `HealthReport` 协议扩展（pressure/error_rate/uptime），Session 健康状态存储，`HealthAwareResolver` 将健康数据注入调度决策
- **模块 H：SDK Proto 同步** — `scripts/sync-proto.sh` 自动同步 `.proto` 定义到各语言 SDK，CI 校验确保版本对齐

### P2 — 技术债清理与扩展能力

- **模块 I：Relay 数据结构优化** — `entries` 内层从 `Vec<RuneEntry>` 改为 `HashMap<String, RuneEntry>`（caster_key→entry），`caster_index` 反向索引加速 `remove_caster` 至 O(R_this_caster)，`generation` 原子计数器支持读路径一致性校验
- **模块 K：Error 类型细化 + Axum 路径语法** — 新增 `RateLimited`/`CircuitOpen`/`Unauthorized`/`Forbidden` Error variant，错误响应统一携带 `request_id`，Axum 0.7→0.8 升级 + 路由从 `:param` 迁移到 `{param}` 语法
- **模块 L：Pilot & 自动扩缩容** — `LoadReport`/`ScaleSignal` 协议扩展，Runtime scaling evaluation loop，`rune pilot daemon` CLI 命令，SDK pilot client 自动发现

### Production Hardening

- **Linus Review** — 18 个 issue 修复：HealthAware `ptr::eq`→值匹配、LeastLoad NaN→`total_cmp`、ScaleEvaluator NaN sentinel→`Option`、KeyCache 三阶段驱逐、CAS task completion、session `list_caster_ids` 逻辑修正、`static` counter→instance field 等
- **handle_session 重构** — 316 行 God Function → `SessionContext` struct + 7 个提取方法，主循环压缩至 ~48 行
- **Telemetry 边界 Sanitization** — `apply_health_report` 为唯一转换边界，pressure/error_rate clamp + pressure_reported 解耦，metrics 非 finite entry 过滤

### 未纳入本版本

- **模块 J：去掉 async_trait** — 推迟至 Rust `async fn in dyn Trait`（RFC 3745）稳定后。当前 `RuneInvoker`/`StreamRuneHandler`/`KeyVerifier` 均通过 `Arc<dyn T>` 使用，原生 async fn in trait 尚不支持 dyn-safe 场景

## Migration Notes

### 配置变更

1. **新增 `[retry]` 配置段**（默认启用）。如需关闭：
   ```toml
   [retry]
   enabled = false
   ```

2. **新增 `[retry.circuit_breaker]`**（默认启用）。断路器按 Caster 粒度隔离，断路时返回 503 CIRCUIT_OPEN。

3. **`[store]` 新增字段**：
   - `reader_pool_size = 4` — 读写分离连接池
   - `key_cache_ttl_secs = 60` — Key 正查缓存
   - `key_cache_negative_ttl_secs = 30` — Key 负查缓存

4. **`[rate_limit]` 新增字段**：
   - `default_caster_max_concurrent = 1024`
   - `per_rune` table 支持 per-Rune 限流

5. **`[auth]` 新增**：`RUNE_AUTH__INITIAL_ADMIN_KEY` 环境变量（仅环境变量，不写入 TOML）

### API 变更

1. **错误响应新增 `request_id` 字段**：
   ```json
   {"error": {"code": "NOT_FOUND", "message": "...", "request_id": "req-abc123"}}
   ```

2. **新增错误码**：`CIRCUIT_OPEN`(503)、`FORBIDDEN`(403)、`TIMEOUT`(504)、`CANCELLED`(499)、`EXECUTION_FAILED`(500)

3. **新增 Flow 错误码**：`STEP_TIMEOUT`、`STEP_RATE_LIMITED`、`STEP_CIRCUIT_OPEN`、`STEP_FORBIDDEN`、`STEP_UNAVAILABLE`、`STEP_UNAUTHORIZED`、`DAG_ERROR`、`NO_TERMINAL_STEP`、`SERIALIZATION_FAILED`

### Axum 升级

Axum 从 0.7 升级到 0.8。路由语法从 `:param` 迁移到 `{param}`。仅影响直接拼路由的外部工具；SDK 用户和 CLI 用户无感。如有自定义中间件或 handler，参考 [Axum 0.8 changelog](https://github.com/tokio-rs/axum/blob/main/axum/CHANGELOG.md)。
