# Rune v0.6.0: Production-Ready — Observability, Rate Limiting & Advanced Scheduling

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox syntax for tracking.

**Goal:** Make Rune production-ready with two modules: (1) observability and operational infrastructure — structured JSON logging, enhanced metrics (success rate, P95 latency), enriched status/caster APIs, per-key rate limiting, and graceful shutdown; (2) advanced scheduling — random and least_load strategies, label-based routing, and priority-based dispatch with fallback.

**Architecture:** Module 9 (Observability & Ops) layers onto existing Gate and Core. Rate limiting is a new middleware in `rune-gate`. Structured logging configures `tracing-subscriber` JSON layer. Graceful shutdown orchestrates SIGTERM handling across gRPC and HTTP servers. Module 10 (Advanced Scheduling) extends the `Resolver` trait and `Relay` in `rune-core` with new strategies, label filtering, and priority sorting.

**Tech Stack:**
- Rust, `tracing` + `tracing-subscriber` (JSON layer), `axum` middleware (rate limiting), `tokio::signal` (graceful shutdown)
- `dashmap` (rate limit counters), existing `Resolver` trait (new implementations)

**Spec:** `docs/superpowers/specs/2026-03-22-rune-v1.0.0-design.md` — Modules 9, 10

```rust
pub struct RandomResolver;

impl Resolver for RandomResolver {
    fn pick<'a>(&self, _rune_name: &str, candidates: &'a [RuneEntry]) -> Option<&'a RuneEntry> {
        if candidates.is_empty() { return None; }
        use rand::Rng;
        let idx = rand::thread_rng().gen_range(0..candidates.len());
        Some(&candidates[idx])
    }
}
```

Add `rand` dependency to `rune-core/Cargo.toml`.

- [ ] **Step 3: Implement `LeastLoadResolver`**

Needs access to `SessionManager` to check `available_permits` per caster. Use a shared reference:

```rust
pub struct LeastLoadResolver {
    session_mgr: Arc<SessionManager>,
}

impl Resolver for LeastLoadResolver {
    fn pick<'a>(&self, _rune_name: &str, candidates: &'a [RuneEntry]) -> Option<&'a RuneEntry> {
        // For local invokers (caster_id=None), always available
        // For remote: pick the one with most available permits (least load)
        candidates.iter().max_by_key(|e| {
            match &e.caster_id {
                None => usize::MAX, // local always preferred
                Some(id) => self.session_mgr.available_permits(id),
            }
        })
    }
}
```

Add `available_permits(&self, caster_id: &str) -> usize` to `SessionManager`.

pub fn resolve_with_labels(

    rune_name: &str,
    required_labels: &HashMap<String, String>,
    resolver: &dyn Resolver,

    let entries = self.find(rune_name)?;
    let filtered: Vec<&RuneEntry> = entries.value().iter()
        .filter(|e| required_labels.iter().all(|(k, v)| e.config.labels.get(k) == Some(v)))
        .collect();
    if filtered.is_empty() { return None; }
    // Use resolver on filtered candidates
    resolver.pick(rune_name, /* ... */)
}
