# Rune v0.1.0 Phase 1: Correctness Ground Truth

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the two critical correctness bugs in SessionManager (permit double-return, heartbeat not evicting) and establish a unit test baseline that locks these behaviors.

**Architecture:** Fix bugs with minimal code changes in `handle_session`. Extract permit-return logic into a helper method `complete_request()` that atomically binds `pending.remove()` with `add_permits(1)`. For heartbeat eviction, use a `tokio::sync::watch` channel to signal the main loop to break. Tests use a helper that constructs a `CasterState` directly and drives `execute()` + manual pending manipulation to verify permit accounting.

**Tech Stack:** Rust, tokio, dashmap, Semaphore

**Spec:** `docs/scope/v0.1.0.md` — Sections "1. Must-Fix Correctness Issues" and "Acceptance Tests > Session / Protocol"

### Task 1: Fix Permit Double-Return Bug

**Files:**
- Modify: `runtime/rune-core/src/session.rs:141-158` (Result handler)
- Modify: `runtime/rune-core/src/session.rs:170-184` (StreamEnd handler)

The bug: `add_permits(1)` executes unconditionally at line 144/173 BEFORE `pending.remove()`. When a request has already been cleaned up by timeout, the late Result/StreamEnd returns a phantom permit, inflating `max_concurrent`.

- [ ] **Step 1: Fix Result handler — move add_permits inside pending.remove**

In `session.rs`, replace lines 141-158 (the `Result` match arm):

```rust
// BEFORE (buggy):
Some(session_message::Payload::Result(result)) => {
    let req_id = result.request_id.clone();
    if let Some(session) = caster_id.as_ref().and_then(|id| self.sessions.get(id)) {
        session.semaphore.add_permits(1);  // BUG: unconditional
    }
    if let Some((_, p)) = pending.remove(&req_id) {
        // ... process result
    }
}

// AFTER (fixed):
Some(session_message::Payload::Result(result)) => {
    let req_id = result.request_id.clone();
    if let Some((_, p)) = pending.remove(&req_id) {
        if let Some(session) = caster_id.as_ref().and_then(|id| self.sessions.get(id)) {
            session.semaphore.add_permits(1);
        }
        let outcome = match result.status() {
            Status::Completed => Ok(Bytes::from(result.output)),
            _ => Err(RuneError::ExecutionFailed {
                code: result.error.as_ref().map(|e| e.code.clone()).unwrap_or_default(),
                message: result.error.as_ref().map(|e| e.message.clone()).unwrap_or_default(),
            }),
        };
        match p.tx {
            PendingResponse::Once(tx) => { let _ = tx.send(outcome); }
            PendingResponse::Stream(tx) => { let _ = tx.send(outcome).await; }
        }
    }
    // Late result (already timed out / cancelled): silently ignored
}
```

- [ ] **Step 2: Fix StreamEnd handler — same pattern**

Replace lines 170-184:


### Task 2: Fix Heartbeat Timeout Forced Eviction

**Files:**
- Modify: `runtime/rune-core/src/session.rs:50-77` (handle_session heartbeat section)

The bug: heartbeat timeout only `warn + break` from the heartbeat background task. The main message loop (`while let Ok(Some(msg)) = inbound.message().await`) keeps running until the gRPC connection closes on its own, which can take seconds. We need the heartbeat task to actively kill the session.

- [ ] **Step 1: Add a shutdown signal channel**

At the start of `handle_session`, after `let semaphore = ...` (line 59), add a `watch` channel:

```rust
let (shutdown_tx, mut shutdown_rx) = tokio::sync::watch::channel(false);
```

- [ ] **Step 2: Heartbeat task sends shutdown signal**

Modify the heartbeat task (lines 64-77). When heartbeat times out, send shutdown signal AND drop the outbound channel:

```rust
let hb_tx = outbound_tx.clone();
let hb_last = Arc::clone(&last_heartbeat_ms);
let hb_shutdown = shutdown_tx.clone();
let hb_handle = tokio::spawn(async move {
    loop {
        time::sleep(HEARTBEAT_INTERVAL).await;
        let msg = SessionMessage {
            payload: Some(session_message::Payload::Heartbeat(Heartbeat { timestamp_ms: now_ms() })),
        };
        if hb_tx.send(msg).await.is_err() { break; }
        let elapsed = now_ms().saturating_sub(hb_last.load(Ordering::Relaxed));
        if elapsed > HEARTBEAT_TIMEOUT.as_millis() as u64 {
            tracing::warn!(elapsed_ms = elapsed, "heartbeat timeout, forcing session shutdown");
            let _ = hb_shutdown.send(true);
            break;
        }
    }
});
```

- [ ] **Step 3: Main loop listens for shutdown signal**

Replace the main loop (line 103) with a `tokio::select!` that also listens for shutdown:

```rust

    let msg = tokio::select! {
        result = inbound.message() => {
            match result {
                Ok(Some(msg)) => msg,
                _ => break,
            }
        }
        _ = shutdown_rx.changed() => {
            tracing::info!("session shutdown signal received");
            break;
        }
    };

    match msg.payload {
        // ... existing match arms unchanged
    }
}
```

- [ ] **Step 2: Write test — normal completion returns permit**

```rust
#[cfg(test)]
mod tests {
    use super::*;
    use tokio::sync::Semaphore;

    fn setup_session(max_concurrent: usize) -> (Arc<SessionManager>, String, Arc<Semaphore>, Arc<DashMap<String, PendingRequest>>) {
        let mgr = Arc::new(SessionManager::new());
        let (tx, _rx) = mpsc::channel(16);
        let semaphore = Arc::new(Semaphore::new(max_concurrent));
        let pending = Arc::new(DashMap::new());
        let caster_id = "test-caster".to_string();

        mgr.sessions.insert(caster_id.clone(), CasterState {
            outbound: tx,
            pending: Arc::clone(&pending),
            semaphore: Arc::clone(&semaphore),
        });

        (mgr, caster_id, semaphore, pending)
    }

    #[tokio::test]
    async fn test_execute_returns_permit_on_normal_completion() {
        let (mgr, caster_id, sem, pending) = setup_session(2);
        assert_eq!(sem.available_permits(), 2);

        // execute() consumes one permit
        let handle = {
            let mgr = Arc::clone(&mgr);
            let cid = caster_id.clone();
            tokio::spawn(async move {
                mgr.execute(&cid, "r-1", "echo", Bytes::from("hello")).await
            })
        };

        // Wait for pending entry to appear
        tokio::time::sleep(Duration::from_millis(10)).await;
        assert_eq!(sem.available_permits(), 1);
        assert!(pending.contains_key("r-1"));

        // Simulate result arriving: remove from pending + return permit
        if let Some((_, p)) = pending.remove("r-1") {
            sem.add_permits(1);
            match p.tx {
                PendingResponse::Once(tx) => { let _ = tx.send(Ok(Bytes::from("world"))); }
                _ => panic!("expected Once"),
            }
        }

        let result = handle.await.unwrap().unwrap();
        assert_eq!(result, Bytes::from("world"));
        assert_eq!(sem.available_permits(), 2);
    }
}
```

```rust
    async fn test_timeout_returns_permit_exactly_once() {
        let (mgr, caster_id, sem, pending) = setup_session(2);

        let handle = {
            let mgr = Arc::clone(&mgr);
```
