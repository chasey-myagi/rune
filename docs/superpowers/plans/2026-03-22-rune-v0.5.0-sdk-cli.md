# Rune v0.5.0: TypeScript SDK & CLI Tool Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox syntax for tracking.

**Goal:** Deliver two developer-facing tools: (1) a TypeScript SDK (`@rune-sdk/caster`) that matches the Python SDK in capability — gRPC bidirectional streaming, heartbeat, reconnection, cancel, file attachments, and full type safety; (2) a Rust CLI (`rune-cli`) for managing Rune Runtime via HTTP API — daemon lifecycle, key management, rune invocation, flow management, logs, and configuration.

**Architecture:** Two independent modules built in parallel: Module 6 is a pure TypeScript npm package using `@grpc/grpc-js` for bidirectional streaming with the Runtime, with proto code pre-generated and embedded; Module 7 is a Rust crate in the existing Cargo workspace using `clap` for command parsing and `reqwest` for HTTP communication with the Runtime's Gate API.

**Tech Stack:**
- Module 6: TypeScript, `@grpc/grpc-js`, `@grpc/proto-loader`, `grpc-tools` (proto codegen), vitest (testing)
- Module 7: Rust, `clap` (CLI parsing), `reqwest` (HTTP client), `tokio` (async), `serde_json` (data), `tabled` (table output)

**Spec:** `docs/superpowers/specs/2026-03-22-rune-v1.0.0-design.md` — Modules 6, 7
